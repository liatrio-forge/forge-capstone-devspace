## Why this matters

The `devspace` binary and the `devspace-tui` companion are installed and
resolved **independently** (adjacent binary, `$DEVSPACE_HOME/bin`, or PATH —
see `findTUIBinary` in `internal/devspace/ui.go:58-73`), so version skew
between them is a normal condition, not an edge case. Today nothing defends
against it: `tui/src/protocol.ts` exports `PROTOCOL_VERSION = 1` but **no code
ever checks it** against the server's `hello.protocol`, and the TS DTO mirror
of the Go structs is kept in lockstep purely by the CLAUDE.md convention
("Protocol DTO changes must update `tui/src/protocol.ts` in lockstep"). A
drifted field silently renders as blank cells or a hung request. After this
plan: (a) a protocol-version mismatch produces one clear error and a clean
exit instead of undefined behavior, and (b) checked-in golden JSON fixtures
are verified by **both** the Go test suite and the Bun test suite, so DTO
drift fails CI on whichever side changed.

## Steps

### Step 1: Go-side fixture generator + verifier

Create `internal/devspace/ui_protocol_fixtures_test.go`. Shape:

- A package-level flag: `var updateUIFixtures = flag.Bool("update-ui-fixtures", false, "rewrite tui/test/fixtures")`.
- A single test `TestUIProtocolFixtures` that builds one **fully-populated**
  value per wire shape (every optional/omitempty field set, so the fixture
  shows the complete shape):
  - `hello.json` — `uiHello{Protocol: uiProtocolVersion, Version: "9.9.9", WorkspaceRoot: "/ws", MachineID: "m-1", MachineName: "mach", SyncMode: "git", Watch: true}`
  - `snapshot.json` — `uiSnapshot` with 2 rows (one with all fields set incl.
    `Branch`, one minimal), a populated `Summary`, a `*Plan` with one `safe`
    and one `skipped` `PlanAction` (with `Reason` and `Project` set) and one
    warning, and a `*Project` with `Remote`, `DefaultBranch`, `EnvProfiles`
    set.
  - `sync-status.json` — `uiSyncStatus` with every field non-zero.
  - `event-watch-refresh.json` — the exact `map[string]any` shape from
    `watchLoop` wrapped in `uiServerEvent{Method: "event", Params: ...}`.
  - `event-watch-error.json` — `uiServerEvent` wrapping
    `{type: "watch-error", message: "boom"}`.
  - `response-error.json` — `uiServerResponse{ID: 7, Error: &uiServerError{Message: "nope"}}`.
- Marshal each with `json.MarshalIndent(v, "", "  ")` + trailing newline.
  Resolve the fixtures dir relative to the source file:
  `filepath.Join("..", "..", "tui", "test", "fixtures")` (package dir is
  `internal/devspace`). With `-update-ui-fixtures`, write the files
  (`os.MkdirAll` + `os.WriteFile`); otherwise read each file and
  `bytes.Equal` against the fresh marshal, failing with a diff hint:
  `"fixture %s is stale; run: go test ./internal/devspace -run TestUIProtocolFixtures -update-ui-fixtures"`.

Run the generator once to create the fixtures:

**Verify**: `go test ./internal/devspace -run TestUIProtocolFixtures -update-ui-fixtures` → pass, then `ls tui/test/fixtures` → 6 files; re-run without the flag → pass.

### Step 2: TS-side validators

In `tui/src/protocol.ts`, add small runtime validators (type guards) for the
shapes the client actually consumes — keep them dumb and explicit, no schema
library (repo has zero runtime deps beyond opentui/react; do not add one):

```ts
export function isHello(v: unknown): v is Hello { ... }
export function isSnapshot(v: unknown): v is Snapshot { ... }
export function isSyncStatus(v: unknown): v is SyncStatus { ... }
export function isServerEvent(v: unknown): v is ServerEvent { ... }
```

Each checks `typeof` of every required field and, for `Snapshot`, the array
element shapes plus optional `plan`/`project` when present (`actions` and
`warnings` may be `null` — the Go `Plan` marshals empty slices as `null`;
`protocol.ts:60-61` already types them `| null`).

**Verify**: `cd tui && bun run typecheck` → exit 0.

### Step 3: TS-side contract test

Create `tui/test/protocol.test.ts`:

- Load each fixture with `Bun.file(new URL("./fixtures/hello.json", import.meta.url)).json()`
  (or `import hello from "./fixtures/hello.json"` — Bun supports JSON imports;
  pick one style and use it for all six).
- `hello.json` → `expect(isHello(data)).toBe(true)` and
  `expect(data.protocol).toBe(PROTOCOL_VERSION)` — this line is the version
  lockstep: bumping `uiProtocolVersion` in Go without bumping
  `PROTOCOL_VERSION` in TS fails this test.
- `snapshot.json` → `isSnapshot` true; additionally assert the two fields the
  UI renders per row exist with the right types on both fixture rows.
- `sync-status.json` → `isSyncStatus` true.
- `event-watch-refresh.json` / `event-watch-error.json` → unwrap
  `.params` and assert `isServerEvent` true.
- `response-error.json` → assert shape `{ id: number, error: { message: string } }`.
- Negative case: mutate a copy of the hello fixture (delete `workspaceRoot`)
  and assert `isHello` returns false — proves the guards actually check.

**Verify**: `cd tui && bun test` → all pass including the new file.

### Step 4: Enforce the handshake in the client

In `tui/src/app.tsx`, replace the hello call (line 70):

```ts
client.request("hello").then(setHello, () => {});
```

with a version check. Target shape:

```ts
client.request("hello").then(
  (hello) => {
    if (hello.protocol !== PROTOCOL_VERSION) {
      fatal(
        `devspace ui-server speaks protocol v${hello.protocol}, this devspace-tui expects v${PROTOCOL_VERSION}` +
          ` (server version ${hello.version ?? "unknown"}). Update devspace and devspace-tui to matching releases.`,
      );
      return;
    }
    setHello(hello);
  },
  (err: Error) => fatal(`hello failed: ${err.message}`),
);
```

`fatal` must exit **cleanly**: the terminal has to be restored before the
message prints, or it's invisible/garbled. Implement it in `main.tsx` (which
owns the renderer): extend `quit` to accept an optional message, e.g.
`quit(code = 0, message?: string)` — after `renderer.destroy()`, if `message`
is set, `process.stderr.write("devspace-tui: " + message + "\n")`, then exit.
Thread it into `App` via the existing `quit` prop (change the prop type to
`(message?: string) => void` mapping to `quit(1, message)` for the fatal
path — keep the plain quit path exiting 0). Keep the diff minimal; do not
build an error screen.

Note: `hello` errors were previously swallowed on purpose (header shows "…").
This step makes hello failure fatal — that is the intended behavior change:
a client that can't hello has no working session.

**Verify**: `cd tui && bun run typecheck && bun test` → exit 0, all pass.
