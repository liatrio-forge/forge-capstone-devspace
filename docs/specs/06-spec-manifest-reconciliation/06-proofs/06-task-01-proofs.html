Task 01 Proofs - Last-synced base manifest is tracked on every successful sync

Sequence 06 · Feature: manifest-reconciliation · Task 1.0 · Phase 3

Task Summary

This task added the base-manifest snapshot: after every successful manifest sync — git workspace push/pull and hosted push/pull — DevSpace now persists the exact manifest content that was synced to <app home>/last-synced-manifest.json (atomic write, 0600). A loader (loadBaseManifest) distinguishes "no base recorded yet" from real errors. This snapshot is the common ancestor that makes true three-way conflict detection possible in Tasks 2.0/3.0: without it, "changed on both sides" is indistinguishable from "independently added on both sides".

What This Task Proves

  • A successful git push and pull each record a base snapshot equal to the manifest that was synced (the normalized/localized form sync actually wrote, which strips machine-local values).
  • A successful hosted push and pull record the same snapshot, alongside the existing hosted version/hash bookkeeping.
  • The loader cleanly signals absence (ok=false, nil error) on a fresh or upgraded install — the documented trigger for two-way fallback mode.
  • The snapshot is written with app-home state permissions (0600) and contains manifest metadata only.
