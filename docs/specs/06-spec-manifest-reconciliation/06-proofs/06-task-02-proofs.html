Task 02 Proofs - Three-way reconcile engine and `workspace reconcile` command

Sequence 06 · Feature: manifest-reconciliation · Task 2.0 · Phase 3

Task Summary

This task turned git-backend manifest divergence from a dead-end error into a reviewable merge. A new devspace workspace reconcile command fetches and localizes the remote manifest (reusing the workspace diff machinery), three-way merges it against the Task 1.0 base snapshot via the pre-existing mergeManifests engine, and writes a review-first plan to DEVSPACE_HOME/last-reconcile.json. Apply is opt-in (--apply), hash-guarded, and backs up the pre-reconcile manifest to DEVSPACE_HOME/manifest-backup.json first. Genuine conflicts always block apply until --force-local/--force-remote resolves them explicitly. With no base snapshot, reconcile degrades to a documented two-way mode where every same-key difference is a conflict.

What This Task Proves

  • One-sided adds/removes/changes auto-merge; change-both, add-add-different, and change-vs-remove are conflicts that are never auto-resolved.
  • The existing engine's delete-vs-modify behavior was corrected: an unchanged-side deletion now merges (removal wins), while a changed-vs-deleted record is a conflict — previously the modified copy silently survived.
  • Divergent-but-mergeable manifests reconcile to the union, with backup + hash guard + base refresh on apply, and a repeat reconcile reports zero ops (idempotent).
  • A genuine conflict blocks --apply (manifest untouched, non-zero exit) and each force flag resolves it to the chosen side.
  • Plain pull/push behavior is unchanged — reconcile is a separate opt-in command.

Reconcile computes a three-way merge between the last synced base manifest, the local
manifest, and the configured Git remote manifest.

By default it writes only a review artifact to DEVSPACE_HOME/last-reconcile.json and does
not change the workspace manifest.

With --apply, DevSpace writes DEVSPACE_HOME/manifest-backup.json before replacing the
local manifest. The base snapshot refreshes after the merged manifest is pushed. Apply is guarded by the local manifest
hash captured when reconcile was generated.

If no base snapshot exists, reconcile falls back to a conservative two-way union:
one-sided additions merge, same-key differences become conflicts.

Use --force-local or --force-remote to resolve every conflict to that side before applying.

USAGE
  devspace workspace reconcile [--flags]
