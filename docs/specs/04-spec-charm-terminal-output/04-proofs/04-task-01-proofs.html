Task 01 Proofs - Charm foundation: dependencies, Liatrio theme, and color-profile plumbing

Sequence 04 · Feature: charm-terminal-output · Task 01 · Phase 3

Task Summary

This task added the Charm v2 dependency set (charm.land/lipgloss/v2, log/v2, fang/v2, huh/v2, bubbles/v2, bubbletea/v2, plus github.com/charmbracelet/colorprofile) and created internal/devspace/styles.go: a shared Liatrio-brand terminal theme and the color-capability detection layer every later restyling task builds on. The theme always renders full-color styles; plain output is produced by wrapping every writer with styledWriter, which downsamples or strips ANSI according to the detected color profile. This means piped output, NO_COLOR, and --no-color all converge on the same code path (the writer), rather than needing parallel plain/colored branches in every future helper.

What This Task Proves

  • Only charm.land/.../v2 import paths were added for the Charm libraries (no v1 github.com/charmbracelet/{lipgloss,bubbletea,bubbles,log,huh,fang} paths anywhere in go.mod), satisfying the spec's v2-only import discipline requirement.
  • The color-profile detection layer resolves once per invocation via colorprofile.Detect, honors NO_COLOR/CLICOLOR/CLICOLOR_FORCE, and the new root --no-color flag forces plain output even when color would otherwise be forced on.
  • Existing commands are unaffected: devspace status still runs correctly with the new PersistentPreRunE wiring in place, and produces zero ANSI bytes today (expected, since no command output has been restyled yet — that is task 3.0's scope).

Evidence Summary

  • Dependency diff: go.mod gains only charm.land/.../v2 and colorprofile entries.
