Notes

  • Tests live beside implementation in internal/devspace/ as *_test.go, isolate state with t.Setenv("DEVSPACE_HOME", t.TempDir()), and are named for the behavior validated (e.g. TestDoctorReportsMissingManifest).
  • Run make verify (test + vet + lint + vulncheck + build) before every commit; golangci-lint is pinned to v2.12.2 with gosec enabled.
  • .golangci.yml excludes fmt.Fprint* from errcheck — new output helpers may keep that idiom.
  • Import discipline: charm.land/.../v2 only; never mix with v1 github.com/charmbracelet/... paths (colorprofile, x/exp/teatest/v2 (test-only), and x/term are exceptions — they live outside the v2 vanity namespace).
  • Conventional-commit prefixes (feat:, fix:, docs:, refactor:); PRs must include terminal output/screenshots since CLI output changes (AGENTS.md).
  • Piped/non-TTY output is the default in tests (buffers are non-TTY), so plain-text assertions require no TTY simulation; styled-path tests force a profile explicitly.
