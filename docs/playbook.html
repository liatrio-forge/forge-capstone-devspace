Interactive CLI walkthrough

Move a workspace manifest between machines without touching project files.

Set your paths once, copy the commands you need, and use the checklist to keep the safe model intact: scan, push the manifest, pull it elsewhere, plan, apply, then hydrate only the project you choose.

Safe by default

            

              DevDrop creates missing folders and placeholders. It does not
              delete local files, overwrite non-empty folders, or pull project
              repos during apply.
            

          

          

            
Stop on drift

            

              If the manifest changed since planning, apply refuses to run.
              Re-run plan so the saved hash matches reality.
            

          

          

            
No secret sync

            

              Git-backed sync stores only manifest.json. Local state, .env
              files, encrypted profiles, and workspace contents stay out of the
              manifest repo.
